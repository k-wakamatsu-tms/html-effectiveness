git push すると何が起きるか

acme/web のデプロイパイプラインを、.github/workflows/ と Argo のマニフェストをもとに図示しています。各ステップをクリックすると、実行内容や所要時間の目安、途中で処理が打ち切られる箇所を確認できます。

処理ステップ 判定 正常終了 失敗経路
図の中のステップをクリック →

git push main

トリガー · 0秒

main へのプッシュまたはマージが deploy ワークフローを起動します。他のブランチへのプッシュは CI で停止し、イメージビルドまで到達しません。

on:
  push:
    branches: [main]
